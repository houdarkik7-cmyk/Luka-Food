# Luka Food Menu — AGENTS.md

## Stack
- pnpm workspaces, Node.js 24+, TypeScript 5.9
- API: Express 5 (bundled via esbuild into `dist/index.mjs`)
- DB: PostgreSQL 16 + Drizzle ORM (node-postgres driver)
- Validation: Zod v4 (`import { z } from "zod/v4"`) — NOT `zod` default
- Frontend: React 19 + Vite + Tailwind CSS 4 + shadcn/ui + React Query + Wouter
- API client: auto-generated by Orval from `lib/api-spec/openapi.yaml`

## Commands
| Action | Command |
|--------|---------|
| typecheck all | `pnpm run typecheck` (runs `tsc --build` for libs then per-package) |
| build all | `pnpm run build` (typecheck → `pnpm -r --if-present run build`) |
| run API dev | `pnpm --filter @workspace/api-server run dev` |
| run Frontend | `pnpm --filter @workspace/luka-food run dev` (requires `PORT` + `BASE_PATH` env) |
| codegen | `pnpm --filter @workspace/api-spec run codegen` |
| push DB schema | `pnpm --filter @workspace/db run push` |
| push DB force | `pnpm --filter @workspace/db run push-force` |

## Monorepo layout
```
lib/
  db/             @workspace/db       — Drizzle schema + pg client (exports db, pool, tables)
  api-zod/        @workspace/api-zod   — Zod schemas (codegen output + manual re-exports)
  api-client-react/ @workspace/api-client-react — React Query hooks (Orval output)
  api-spec/       @workspace/api-spec  — OpenAPI 3.1 spec + Orval config
artifacts/
  api-server/     @workspace/api-server — Express 5 app (entry: src/index.ts → app.ts → routes/)
  luka-food/      @workspace/luka-food  — React SPA (Vite dev on :8080)
```

## Key architecture facts
- **API routes** are registered in `artifacts/api-server/src/routes/index.ts` (categories, menuItems, orders, health). Each is a separate file.
- **Zod schemas** are generated from `lib/api-spec/openapi.yaml` via Orval. After changing the OpenAPI spec, run codegen before touching route files.
- **Price column** is `numeric(10,3)` in Postgres (Tunisian Dinar, 3 decimal places). Drizzle returns it as a string — always `parseFloat()`.
- **Images** are stored as base64 data URLs in `image_url` column. Express JSON limit is 10 MB (`express.json({ limit: "10mb" })`).
- **CASCADE deletes** are set on Drizzle schema `{ onDelete: "cascade" }` AND must match actual DB foreign keys. Currently: `menu_items.category_id → categories.id` and `order_items.menu_item_id → menu_items.id`.
- **Order total** is computed server-side from live menu item prices at creation time.
- **Client app** (`/`): browse by category, cart, order by table number. **Kitchen** (`/kitchen`): order monitor + menu management (CRUD categories & items).
- **Dark mode** forced: `<html class="dark">`. RTL Arabic: `dir="rtl"`, Cairo font.

## Windows quirks
- Use `pnpm.cmd` instead of `pnpm` (PowerShell `ExecutionPolicy` blocks `.ps1`).
- The `pnpm-workspace.yaml` **overrides** block excludes `win32-x64` esbuild — keep that or esbuild install fails.
- `drizzle.config.ts` uses relative path `"./src/schema/index.ts"` (not `__dirname` + join) to work on Windows.
- Vite config proxies `/api` → `http://localhost:5000` (required for dev on Windows without Replit proxy).
- Run servers in separate PowerShell windows. Stop with `Ctrl+C`.

## Error handling
- `413` for oversized payloads — body parser middleware before routes.
- `500` generic — catches unexpected errors, returns `{ error: "..." }` (Arabic).
- All route handlers validate with Zod before DB access — `400` on parse failure.
- Delete endpoints (`categories/:id`, `menu-items/:id`) return `204` (no content) on success, `404` on not found.
